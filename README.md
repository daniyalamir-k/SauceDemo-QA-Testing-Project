# SauceDemo QA Testing Project

This project demonstrates Quality Assurance testing of the SauceDemo web application, including Manual Testing, Functional Testing, UI Validation, Bug Reporting, and basic Performance Analysis to ensure application reliability and usability.

## Project Overview
This project focuses on end-to-end manual and performance testing of the SauceDemo (Swag Labs) e-commerce application. The objective was to validate functional behavior and identify performance issues under different user scenarios.

## Tools & Environment
- Testing Types: Manual Testing, Functional Testing, Negative Testing, Performance Testing  
- Tools: Chrome DevTools, MS Excel (Test Cases), MS Word (Bug Reports)  
- Environment: Windows 11 | Google Chrome (Latest Version)

## Key Findings (Performance)
- Standard User load time: ~762ms (0.76s)  
- Performance Glitch User load time: ~5.49s  
- Identified high TTFB (Time to First Byte) indicating backend latency issues via Chrome DevTools Network analysis

## Testing Scope
- Functional Testing: Login, Add to Cart, Sort Products, Checkout flow  
- Negative Testing: Invalid credentials, empty fields validation  
- Performance Monitoring: Page load timing and network response analysis

## Deliverables
- Test Cases Document (Positive & Negative scenarios)  
- Bug Reports with severity, priority, and reproduction steps  
- Performance observation using DevTools

## Note
All testing was performed on the SauceDemo demo application for QA learning purposes.
